# Changelog

All notable MatriSense development milestones are tracked below. The version labels are project milestones, not clinical release approvals.

### Version 0.6 — Docs and Evidence Layer
*   Added public `/docs` page for judges and reviewers.
*   Added admin-controlled docs visibility and scheduling.
*   Added live stats and structured documentation sections.
*   Added Markdown-backed content and evidence library planning.

### Version 0.5 — Profile, Records, and Dashboard Improvements
*   Added or improved patient profile editor.
*   Added health worker profile/checker workflow.
*   Added optional patient document and medical report support where available.
*   Added patient list alongside triage case list in health worker dashboard.
*   Improved profile and dashboard wiring.

### Version 0.4 — Regional Referral Work
*   Planned or integrated district/upazila fields.
*   Added seeded hospital lookup and hospital assignment workflow where enabled.
*   Added assignment history and audit integration design.
*   Added location snapshot support for triage sessions where enabled.

### Version 0.3 — Health Worker Workflow
*   Added health worker dashboard.
*   Added worker case detail.
*   Added case status update.
*   Added referral or follow-up note workflow.

### Version 0.2 — AI Triage and Safety
*   Added Bangla symptom extraction.
*   Added confirmation and follow-up questions.
*   Added rule-based danger-sign triage.
*   Added rule-aware RAG guidance.
*   Added LLM Bangla explanation and safety validator.

### Version 0.1 — Core Patient Flow
*   Added patient profile.
*   Added Bangla symptom input.
*   Added patient result page.
*   Added patient triage history foundation.
