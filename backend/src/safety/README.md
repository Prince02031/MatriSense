# MatriSense Safety Layer

## 1. Purpose
The Safety Layer is a non-negotiable medical guardrail that wraps around the AI generation pipeline. It ensures that MatriSense remains strictly a **triage and guidance tool, not a doctor**. It prevents the LLM from hallucinating medical diagnoses, prescribing drugs, or downgrading emergency cases.

## 2. Pre-Generation Check (`validatePreGeneration`)
Validates the internal system state **before** calling the LLM. It checks:
- The rule engine's risk level and allowed guidance type are logically sound.
- `HIGH` risk cases do not accidentally include "self-care" guidance.
- The `careGuidanceContext` is properly populated with verified medical `sources`.
- Mandatory global safety constraints (e.g., `diagnose_disease`, `prescribe_medicine`) are present in `blockedAdvice`.

## 3. Post-Generation Check (`validateLLMOutput`)
Acts as a strict censor on the Bangla text generated by the LLM **before** showing it to the user. It checks:
- No forbidden Bangla or English medical terms are present.
- The LLM did not alter or downgrade the `riskLevel`.
- The LLM's suggested `stepsNowBn` strictly matches the allowed RAG context (no hallucinated advice).
- The output includes the mandatory Bangla safety disclaimer.

## 4. Fallback Templates (`fallbackTemplates.js`)
If either validation step fails, the system **aborts the LLM output** and immediately substitutes a predefined, safe fallback template corresponding to the user's risk level (`LOW`, `MEDIUM`, or `HIGH`). This fail-safe mechanism ensures the user always receives verified guidance, even if the AI misbehaves.

## 5. Examples of Blocked Wording
The validator will block any payload containing phrases like:
- **Diagnoses**: `আপনার রোগ হলো`, `প্রি-এক্লাম্পসিয়া`, `ক্যান্সার`
- **Medicines**: `এই ওষুধ খান`, `প্যারাসিটামল`, `ডোজ`, `ট্যাবলেট`
- **False Reassurance**: `১০০% গ্যারান্টি`, `চিন্তার কিছু নেই`, `ঠিক হয়ে যাবে`
- **Dangerous Dismissals**: `ডাক্তার দরকার নেই`, `হাসপাতালে যাওয়ার দরকার নেই`

## 6. How to Run Tests
Run the standalone safety tests to verify core logic:
```bash
npm run test:safety
```
Run the local post-LLM validation demo:
```bash
npm run test:safety:llm
```
Run the full triage-to-assembly pipeline test:
```bash
npm run test:pipeline
```

## 7. Important Rule 🚨
**The LLM CANNOT:**
- Diagnose a medical condition.
- Prescribe medicine or dosages.
- Downgrade the triage risk level.
- Add home-care advice outside of the retrieved RAG guidance.
